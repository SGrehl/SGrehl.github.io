About Processing Time

The faster your model runs, the better:

  • Enables more experimentation and interactive exploration.

  • Allows for more replications and larger-scale models.

  • Makes parameter tuning and sensitivity analysis feasible.

  • Reduces computational costs (especially important on shared or cloud resources).

Redundancy

  • In run_sequence(), the number of unhappy agents is calculated twice per step:

    # 2) Iterate (without verbose code) 
for (t in seq_len(model$max_t)) {
  unhappy <- get_unhappy_agents(world, model)
  unhappy_series[t] <- length(unhappy) / length(which(V(world)$type != 0))

  segregation_series[t] <- get_data_segregation(world)

  # stop if equilibrium reached
  if (unhappy_series[t] == 0) {
    break
  }

  # otherwise perform one step
  world <- run_step(world, model)
}

    • First, to check for equilibrium

    • Then again, during a time step, for find agents that want to relocate

  • This results in duplicated get_unhappy_agents() calls, a costly operation.

Redundancy - Optimization Strategy

  • Move the unhappiness and segregation calculations into run_step()

      world$data_unhappy     <- length(unhappy) / length(which(V(world)$type != 0))
  world$data_segregation <- get_data_segregation(world)

  • This way:

    • Metrics are computed once per step

    • Data is stored immediately after agent actions

    • No redundant evaluations

Redundancy - Optimization Strategy II

  • Now, we must rearrange run_sequence()

     # 3) Iterate (without verbose code)
  for (t in seq_len(model$max_t)) {
    # Perform one step (and update metrics inside world)
    world <- run_step(world, model)

    # Store the data from world
    unhappy_series[t]     <- world$data_unhappy
    segregation_series[t] <- world$data_segregation 

    # stop if equilibrium reached
    if (unhappy_series[t] == 0) {
      break
    }
  }

Storing fixed values

  • In our model: The neighborhood is fixed
  • Yet get_unhappy_agents and get_data_segregation both calculate it every time step anew:
    • nbrs <- adjacent_vertices(world, V(world))
  • Instead: Calc it once in init_world and store it in world:
    • world$nbrs <- adjacent_vertices(world, V(world))
  • And use world$nbrs in both functions instead.
